Network Token Authentication (Mastercard Passkeys)

Authenticate card-on-file payments with network passkeys, starting with Mastercard Secure Card on File (SCOF), without integrating network-specific SDKs.

What it is

Network Token Authentication (NTA) turns stored-card payments into tokenized, passkey-authenticated transactions. The shopper authenticates once against the network with a device passkey (biometric), and Yuno handles the cryptography and redirects. It is opt-in per merchant and triggered through standard routing rules, with no change to the payment creation call.

  • Higher approval rates and liability shift on authenticated transactions.
  • SCA compliance with lighter friction than standalone 3DS.
  • One integration for every network, controlled per transaction via routing.

How it works at Yuno

NTA runs only when configured. Setup, once per merchant:

  1. Onboard the account in Network Tokens.
  2. Create a connection of type Network Token Authentication in the Dashboard.
  3. Load 3DS information on the payment provider connection.
  4. In routing, place the NTA step before the payment provider step.

At runtime, Yuno evaluates routing before authorization. If the route includes the NTA step it triggers the passkey flow; otherwise the payment proceeds as a standard authorization.

Web integration Confirmed

When NTA is triggered, Yuno surfaces it as a standard REDIRECT_URL continue action, the same pattern as redirect-based APMs. After creating the payment, get the continue action and send the shopper to the network-hosted URL with a full-page redirect:

const data = await apiClientPayment.getContinuePaymentAction({
  checkoutSession: "<CHECKOUT_SESSION>",
})

// data.action === "REDIRECT_URL"
// data.redirect = { init_url, success_url, error_url }

if (data.action === "REDIRECT_URL") {
  window.location.href = data.redirect.init_url
}

After authentication the shopper returns to your site. Read the final status via Webhooks or Retrieve Payment by ID. Full token and Create Payment steps are in the integration reference.

Miguel: The web surface is a full-page REDIRECT_URL. SDK_PROVIDER was deprecated by Mastercard.

Final status

The flow is asynchronous. Wait for the outcome via webhook or polling. Terminal states are standard Yuno statuses: SUCCEEDED, DECLINED, CANCELLED, EXPIRED.

Helio Cantelli May 13, 2026 · 15:04

Feedback: docs.y.uno/docs/payment-features/network-token-authentication

Missing: Passkey UX flow diagrams (Mastercard SCOF)
  • Two distinct scenarios need to be shown:
    • First-time user: 3DS challenge → passkey creation prompt → cryptogram generated
    • Returning user (passkey exists): biometric prompt only, no 3DS, frictionless
  • Reference: Mastercard SCOF passkey docs + internal CEH Mastercard Payment Passkeys page
Missing: Mobile SDK integration (iOS & Android)
  • Current doc only shows web JS (window.location.href) — zero mention of native SDKs
  • Mastercard has no native mobile SDK; Yuno uses a WebView wrapper on both platforms
  • Mobile flow uses continuePayment() + a callback_url deeplink (same pattern as C2P passkey)
  • Should be a dedicated mobile section parallel to the web integration section
Missing: SDK mode coverage
  • Only shows Headless/Seamless (apiClientPayment) — no mention of Full/Rendered or Lite modes
  • Should clarify which modes support NTA and whether continue action handling differs
Missing: Passkey binding rules
  • Passkey is bound to: specific card + specific device + specific browser
  • Switching any one of the three requires creating a new passkey
  • Important for merchants to understand why some returning users still hit 3DS
Missing: Fallback & error behavior
  • No mention of what happens on: user cancels, auth fails, session expires
  • Terminal states (CANCELLED, EXPIRED, NOT_AUTHENTICATED) should be documented with webhook status mappings
Clarification needed: Web action type
  • Public doc shows REDIRECT_URL + full-page redirect (window.location.href)
  • Internal arch page shows web uses a popup (window.open) with postMessage protocol
  • Should clarify: is the merchant surface REDIRECT_URL or SDK_PROVIDER? Or both depending on SDK mode?
Minor
  • Add sandbox/testing guidance (currently none)
  • Mention passkeys created via SCOF are interoperable with C2P (network-level storage)
  • Document authentication_method field in webhook payload (e.g. FIDO2 for passkey, 3DS_ACS for 3DS)
  • "Additional networks" line could note Visa passkeys are in progress
